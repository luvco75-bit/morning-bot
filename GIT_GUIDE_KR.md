# Git 버전 관리 가이드 (초보자용)

## Git이란?

Git은 **"저장"이 아니라 "버전 관리"** 도구입니다.

| 개념 | 설명 |
|------|------|
| **파일 저장** | Ctrl+S 누르면 디스크에 저장됨 (Git 없어도 됨) |
| **버전 관리** | "언제, 뭘, 왜 바꿨는지" 기록을 남기는 것 |

파일을 수정하고 저장하면 `C:\Projects\abeoma-app` 폴더에 바로 반영됩니다.
컴퓨터를 꺼도 파일은 사라지지 않아요.

---

## AI 대화가 사라지는 이유

Claude Code, ChatGPT 등 AI 도구는 **세션 기반**으로 동작합니다.

- 브라우저를 닫거나 새 대화를 시작하면 이전 대화 맥락이 초기화됩니다
- 하지만 AI가 수정한 **코드 파일은 디스크에 그대로 남아있습니다**
- 대화가 사라져도 코드는 안전합니다

### 작업을 이어서 하려면?

1. 새 대화를 열고 프로젝트 폴더에서 시작
2. "이 프로젝트 이어서 작업하자"라고 말하면 AI가 코드를 읽고 파악합니다
3. Git 커밋 메시지를 잘 남겨두면 AI가 이전 작업 흐름을 더 빨리 이해합니다

## Git 기본 흐름 (5단계)

```
수정 → add → commit → push → 끝!
```

### 1단계: 파일 수정
그냥 코드를 수정하고 저장하면 됩니다. (이건 이미 하고 계신 거예요)

### 2단계: 변경 사항 확인
```bash
git status          # 뭐가 바뀌었는지 확인
git diff            # 어떻게 바뀌었는지 상세 확인
```

### 3단계: 스테이징 (add)
```bash
git add app.py              # 특정 파일만 추가
git add .                   # 모든 변경 파일 추가
```

### 4단계: 커밋 (commit)
```bash
git commit -m "텔레그램 봇 메시지 형식 수정"
```
커밋 메시지는 **"뭘 왜 바꿨는지"** 간단히 적으세요.

### 5단계: GitHub에 올리기 (push)
```bash
git push origin main
```

## 자주 쓰는 Git 명령어

| 명령어 | 설명 |
|--------|------|
| `git status` | 현재 상태 확인 |
| `git log --oneline` | 커밋 이력 한 줄씩 보기 |
| `git diff` | 변경 내용 확인 |
| `git add <파일>` | 커밋할 파일 선택 |
| `git commit -m "메시지"` | 변경 사항 기록 |
| `git push origin main` | GitHub에 업로드 |
| `git pull origin main` | GitHub에서 최신 코드 가져오기 |

## 비유로 이해하기

| Git 개념 | 비유 |
|-----------|------|
| **Working Directory** | 작업 책상 (지금 수정 중인 파일들) |
| **Staging Area (add)** | 택배 상자에 넣기 (보낼 것 고르기) |
| **Commit** | 택배 상자 봉인 + 라벨 붙이기 (기록 남기기) |
| **Push** | 택배 발송 (GitHub에 올리기) |
| **Pull** | 택배 수령 (GitHub에서 가져오기) |

## 실수했을 때 복구하기

```bash
# 아직 commit 안 한 변경 사항 되돌리기
git checkout -- app.py

# 마지막 commit 상태로 전체 되돌리기
git reset --hard HEAD

# 이전 commit 이력 보기
git log --oneline

# 특정 commit으로 돌아가기
git checkout abc1234
```

> **주의:** `git reset --hard`는 변경 사항을 완전히 삭제합니다. 신중하게 사용하세요.

## 핵심 요약

1. **파일 저장 ≠ Git** - Ctrl+S로 저장한 파일은 컴퓨터에 안전하게 남아있음
2. **AI 대화는 휘발성** - 대화는 사라져도 코드 파일은 그대로
3. **Git = 타임머신** - 변경 이력을 기록해서 언제든 과거로 돌아갈 수 있음
4. **GitHub = 클라우드 백업** - push하면 컴퓨터가 고장나도 코드가 안전
5. **커밋 메시지를 잘 쓰면** - 다음 AI 세션에서도 작업 흐름을 빠르게 파악 가능
